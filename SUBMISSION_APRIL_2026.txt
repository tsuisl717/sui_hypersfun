4. Work Completed This Month

Core vault contracts (carried from March, refined in April)

The vault system is built around a bonding curve AMM that prices vault tokens against a USDC reserve. Users hold VaultShare NFTs with per-share entry NAV and acquisition timestamps, enabling individual performance fee and time-based exit fee calculation. TWAP-based NAV protection prevents single-block price manipulation. Exit fees step down from 15% to 0% over 30 days.

The factory module handles vault creation, graduation tiers, and global settings (min deposit, max buy limits). Bonding curve math lives in sui_math.move for testability.

DeepBook V3 spot integration (refined April)

Refactored the spot trading flow to use single-PTB atomic execution. The authorize_swap function now returns the SwapAuthorization object (instead of transferring it), so the entire authorize -> consume -> DeepBook swap -> deposit flow executes in one transaction with one wallet signature. Previously required two separate transactions.

Added DeepBook pool price quote fetching in the frontend — shows mid price and estimated output before trade execution.

DeepBook Margin integration (NEW — April)

This is the month's biggest addition. Built a complete margin trading layer that lets Leaders execute leveraged long/short positions through DeepBook Margin — Sui's native margin infrastructure.

New Move module: sui_margin.move

6. Integration Description

DeepBook V3 (spot)

The integration lives in sui_deepbook.move. authorize_swap returns a SwapAuthorization that gets consumed in the same PTB — single transaction, single signature. The full flow: authorize -> consume (extract USDC from vault) -> DeepBook swap_exact_quote_for_base -> deposit_swap_output (deposit base asset to AssetVault). If any step fails, the entire transaction reverts atomically.

AssetVault<T> holds non-USDC assets from swaps (SUI, DEEP, etc.) and the NAV calculation includes their value so token pricing reflects the full portfolio.

DeepBook Margin (leveraged trading)

sui_margin.move manages the authorization layer between the vault and DeepBook Margin. The frontend uses @mysten/deepbook-v3 SDK which handles Pyth oracle price feeds, margin pool routing, and order placement automatically.

The vault authorization ensures:
- Funds can only be extracted with Leader's authorization + time-limited auth objects
- Maximum allocation limits prevent over-exposure (default 50% of vault reserve)
- All funds must be returned to the vault — Leader cannot withdraw to personal wallet
- Cumulative P&L is tracked on-chain per margin account

DeepBook Margin is spot margin (not perpetuals):
- Long: deposit USDC collateral -> borrow more USDC -> buy base asset
- Short: deposit USDC collateral -> borrow base asset -> sell for USDC
- Liquidation: automated on-chain when risk ratio drops below threshold

Seal (MEV protection)

sui_seal.move implements the seal_approve entry function that Seal key servers call to verify decryption access. Uses the Time-Lock Encryption (TLE) pattern where the identity is a BCS-encoded u64 timestamp. The key servers will only release decryption keys after the timestamp has passed.

SealedTradeIntent objects store the encrypted trade data on-chain. The Leader submits their encrypted intent, waits for the time lock, then decrypts and executes.

Walrus (trade history)

Uses Walrus REST API (publisher/aggregator) for storing vault trade history and NAV snapshots as immutable blobs. No SDK dependency — direct HTTP PUT/GET. Each upload returns a blob ID that anyone can use to verify the vault's complete track record. Blob chain (previous blob IDs stored in each upload) provides an audit trail.

zkLogin (social login)

Uses @mysten/enoki for managed zkLogin. Handles salt management, ZK proof generation, and session management. Users authenticate with Google OAuth, get a Sui address derived from their Google identity, and can interact with vaults without any wallet extension. Combined with Enoki's sponsored transactions, the UX is comparable to Web2 fintech apps.
